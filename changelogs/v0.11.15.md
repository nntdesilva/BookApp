# Version 0.11.15 - 2026-03-24

## Refactor: LangGraph agent loop in chat-service

Completes the LangChain migration started in v0.11.14 by replacing the manual agentic while-loop with `createReactAgent` from `@langchain/langgraph/prebuilt`.

### chat-service changes

- **`aiService.js`:** Introduced `createTools(userId)` — a per-request factory that produces 8 `tool()` instances with `userId` bound in each closure and a shared `capturedVisualizationHtml` side-effect variable. The `generate_visualization` tool strips the raw HTML before returning to the LLM and exposes it via `getVisualizationHtml()` instead. `runChatTurn(message, history, userId)` replaces both `generateChatResponse` and `continueAfterFunctionExecution`: it creates the agent with `createReactAgent`, invokes it once, and returns the final response alongside the updated history and any captured visualization. History stored per turn is compacted to `[HumanMessage, AIMessage(final)]` only — no intermediate tool-call messages.
- **`chatRoutes.js`:** Removed the `while (requiresFunctionExecution)` loop, the `executeFunction` switch statement, the per-round error propagation, and all direct imports of `favoritesClient`, `booksClient`, and `analysisClient`. The `/chat` handler now calls `aiService.runChatTurn` once and processes the result.
- **`tests/aiService.test.js`:** Rewritten for the new API. `@langchain/langgraph/prebuilt` is mocked so `createReactAgent` returns a fake agent with a configurable `invoke` spy. A separate `createTools` test group calls each tool function directly, verifying correct client delegation and the visualization HTML capture/strip behaviour.
- **`tests/chatRoutes.test.js`:** Updated to mock `aiService.runChatTurn`. Tool-dispatch tests (which tested route-layer concerns that now live in `aiService`) replaced with focused tests for argument forwarding, visualization passthrough, and the no-visualization case.

Dependencies: `@langchain/langgraph` (already a transitive dep; now used directly).
