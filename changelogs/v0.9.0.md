# Version 0.9.0 - 2026-02-14

## Major Feature: Interactive Data Visualization with Plotly.js

Book analysis results can now be rendered as interactive charts. The AI writes Python code to generate self-contained Plotly.js HTML, which is displayed inline in the chat. Smart session-based caching allows switching between chart types without recomputing the underlying data.

## What's New

### Visualization Tool

A new `generate_visualization` function joins the existing `analyze_book_statistics` tool:

| Tool | Description |
|---|---|
| `generate_visualization` | Generate an interactive Plotly.js chart from book text analysis |

Parameters:

- `bookTitle` ‚Äî the book to analyse
- `question` ‚Äî what data to compute and visualise
- `chartType` ‚Äî the type of chart to create

### Supported Chart Types

- Bar chart
- Pie chart
- Line chart
- Scatter plot
- Heatmap
- Histogram
- Treemap
- Sankey diagram

### Session-Based Caching

When the AI generates a visualization, the computed data is cached in the user's session. If the user asks to see the same data as a different chart type (e.g. "show that as a pie chart"), the data is reused without re-fetching the book or re-running analysis ‚Äî only the chart rendering step runs again.

### Self-Contained HTML

The AI generates Plotly.js visualizations as fully self-contained HTML strings (including the Plotly CDN script tag, data, layout, and config). The HTML is extracted between `---HTML_START---` and `---HTML_END---` markers and rendered inline in the chat interface via an iframe.

## Technical Details

### Modified Files

- **`services/analysisService.js`** (formerly `assistantService.js`) ‚Äî added `analyzeAndVisualize()` function and `VISUALIZATION_SYSTEM` prompt
- **`services/aiService.js`** ‚Äî added `generate_visualization` tool definition
- **`controllers/bookController.js`** ‚Äî added `generate_visualization` case to tool-call dispatch; session caching logic for computed data
- **`views/index.ejs`** ‚Äî HTML rendering for visualization output in chat messages

### Visualization Pipeline

```
User asks visualization question
  ‚Üí AI calls generate_visualization(bookTitle, question, chartType)
    ‚Üí Fetch book text from Gutenberg (or use cached data)
    ‚Üí Upload to Claude Code Execution
    ‚Üí Claude writes Python to compute data + generate Plotly HTML
    ‚Üí Extract HTML between markers
    ‚Üí Render in chat as interactive iframe
```

## Example

```
User: "Show me the top 10 most common words in Pride and Prejudice as a bar chart"

AI Process:
1. Calls generate_visualization("Pride and Prejudice", "top 10 most common words", "bar chart")
2. Fetches book text, uploads to Claude
3. Claude writes Python to count words and build Plotly bar chart HTML
4. Interactive chart rendered inline in chat

User: "Now show that as a pie chart"
‚Üí Reuses cached word count data, only re-renders as pie chart
```
