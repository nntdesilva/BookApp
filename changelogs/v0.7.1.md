# Version 0.7.1 - 2026-02-08

## Bug Fix: Eliminated Intermediary Response Step

This patch fixes a user experience issue where the AI would announce its intentions before executing word count queries, creating an unnecessary two-step interaction.

## What's Fixed

### AI Response Behavior

- **Silent Function Execution**: AI now executes word search functions in the background without announcing intermediate steps
- **Direct Answers**: Users receive immediate results instead of "I will check..." followed by actual results
- **Improved UX**: Single-response flow for word count queries

### Technical Changes

**Modified Files:**

- `services/aiService.js`:
  - Added instruction in system prompt to execute functions silently
  - Updated TWO-STEP PROCESS section to emphasize background execution
  - No change to function logic, only communication behavior

- `services/gutenbergService.js`:
  - Removed debug console.log statement

## Before vs After

### Before (v0.7.0)
```
User: "How many times does 'Uriah' appear in David Copperfield?"
AI: "The book 'David Copperfield' by Charles Dickens is available. Now, I will count how many times 'Uriah' appears in it."
User: [clicks "yes pls"]
AI: "The word 'Uriah' appears 253 times in David Copperfield by Charles Dickens."
```

### After (v0.7.1)
```
User: "How many times does 'Uriah' appear in David Copperfield?"
AI: "The word 'Uriah' appears 253 times in David Copperfield by Charles Dickens."
```

## Impact

- **Response Time**: Perceived as faster due to single interaction
- **User Experience**: More natural, direct communication
- **Functionality**: No change to underlying word count feature
- **Backward Compatible**: No breaking changes

## Files Changed

- `services/aiService.js` (3 lines added)
- `services/gutenbergService.js` (1 line removed)

This minor release improves the conversational flow without affecting any core functionality or requiring configuration changes.
