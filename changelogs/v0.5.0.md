# Version 0.5.0 - 2026-02-02

## Major Refactor: Agentic Architecture

This release represents a significant architectural shift from a multi-service approach to a unified meta-prompt system for handling book information queries.

## What's New

### Meta-Prompt System

- **Unified AI Processing**: Introduced a single, powerful meta-prompt system (`utils/metaPrompt.js`) that consolidates multiple AI operations:
  - Automatic spelling correction
  - Book vs. Series detection
  - Intelligent tagging in responses
  - Topic change detection in follow-up conversations

### Enhanced Tag System

The new system uses three distinct tag types:

- `<original-book>`: For the exact book the user searched for (must have ISBN)
- `<book-in-series>`: For books in a series when user searches for a series name
- `<unrelated-book>`: For books mentioned in follow-up conversations that are unrelated to the original topic

### Intelligent Topic Change Detection

- **Context-Aware Conversations**: System now detects when users change topics during follow-up conversations
- **Smart Association Mapping**: Includes comprehensive databases of:
  - Known authors (188+ well-known authors)
  - Known books and series (70+ popular titles)
  - Author-work associations for relationship detection
- **Confidence Scoring**: Multi-level confidence detection (high, medium, none) for topic changes

### Simplified Controller Architecture

- **Streamlined Controllers**: Refactored `bookController.js` to focus purely on HTTP request handling
- **Clean Separation**: Controller now delegates all AI logic to the meta-prompt service
- **Session Management**: Improved conversation history management

## Removed

### Colored Badge Detector System

- **Removed**: `utils/coloredBadgeDetector.js` and its associated test file
- **Reason**: Replaced by the more sophisticated meta-prompt system that handles tagging natively within AI responses
- **Migration**: All badge color logic is now handled by the meta-prompt's intelligent tagging system

### Documentation Cleanup

- **Removed**: `TESTING_GUIDE.md`
- **Added**: Comprehensive AI flow documentation:
  - `CHAT_AI_FLOW.md`: Detailed sequence diagram for chat functionality
  - `SEARCH_AI_FLOW.md`: Complete search flow documentation

## Technical Details

### Architecture Changes

**Before (v0.4.1):**

```
User → Controller → AI Service (spelling) → Title Correction → Colored Badge Detector → Tag Service → Response
```

**After (v0.5.0):**

```
User → Controller → Meta-Prompt System (unified processing) → Response
```

### Key Benefits

1. **Reduced Complexity**: Eliminated multi-step processing pipeline
2. **Better Context Awareness**: Single AI call maintains better context
3. **Improved Accuracy**: Meta-prompt system provides more consistent tagging
4. **Enhanced Maintainability**: Single source of truth for AI behavior
5. **Topic Change Handling**: New capability to detect and handle conversation topic changes

## Migration Notes

### For Developers

If you're working with the previous version:

1. **Tag System**: The XML tags have changed. Update any code that parses or processes tags:

   - Old: Various badge detection logic
   - New: `<original-book>`, `<book-in-series>`, `<unrelated-book>` tags

2. **AI Service Calls**: Direct calls to spelling correction and title correction are now handled by:

   - `metaPrompt.generateSearchResponse()` for initial searches
   - `metaPrompt.generateChatResponse()` for follow-ups

3. **Badge Colors**: Badge color assignment is now automatic based on tag type:
   - `<original-book>` → Cream badge
   - `<book-in-series>` → Green badge
   - `<unrelated-book>` → Orange badge

### For Users

- **Improved Experience**: More accurate and context-aware book recommendations
- **Better Follow-ups**: System now understands when you change topics in conversation
- **Consistent Tagging**: More reliable book title highlighting

## Breaking Changes

⚠️ **API Changes**: If you were using the colored badge detector utility directly, you'll need to migrate to the meta-prompt system's built-in tagging.

## Performance

- **API Calls**: Reduced from 2-3 AI calls per search to a single unified call
- **Response Time**: Improved due to elimination of multi-step processing
- **Token Usage**: More efficient prompt engineering reduces token consumption
