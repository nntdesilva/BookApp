# Version 0.11.3 - 2026-03-11

## Bug Fix: Missing MONGODB_URI on Auth and Favorites Services

Fixed login and signup failures on AWS App Runner caused by the auth and favorites services never having `MONGODB_URI` set in their runtime environment. Without it both services fell back to `mongodb://127.0.0.1:27017`, hitting Mongoose's 10-second buffering timeout on every database operation and returning a 500 to the gateway on every login and signup attempt.

## What's New

### Root Cause

The App Runner services for `auth-service` and `favorites-service` were created without `MONGODB_URI` in their `RuntimeEnvironmentVariables`. Mongoose queued all operations for 10 seconds waiting for a connection to `localhost:27017` (which does not exist in the App Runner container), then timed out.

### Fix

- Set `MONGODB_URI` and `NODE_ENV=production` directly on both App Runner services via the AWS console
- Added `MONGODB_URI` as a GitHub Actions secret (`secrets.MONGODB_URI`) so the value is available to the CD pipeline
- Updated the `deploy-auth` and `deploy-favorites` steps in `.github/workflows/cd.yml` to pass `MONGODB_URI` in `RuntimeEnvironmentVariables` on every `update-service` call, preventing the env var from being wiped on future deployments
- Added `MONGODB_URI` to the "Verify required secrets" pre-flight check in both deploy jobs so the CD pipeline fails fast if the secret is absent rather than silently deploying a broken container
