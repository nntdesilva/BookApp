# Version 0.6.0 - 2026-02-03

## Major Feature: Favorites List Management

This release introduces a comprehensive favorites list system that allows users to save and manage their favorite books through AI-powered conversations.

## What's New

### Favorites Service

- **Session-Based Storage**: Implemented `services/favoriteService.js` with full CRUD operations:
  - `addFavorite()`: Add books to favorites with ISBN-13 validation
  - `removeFavorite()`: Remove books from favorites by ISBN
  - `listFavorites()`: View all saved favorite books
  - `clearFavorites()`: Clear all favorites from the list
  - `isFavorite()`: Check if a book is already favorited
  - `getFavoriteCount()`: Get total number of favorites

### ISBN-13 Validation

- **Checksum Validation**: Full ISBN-13 validation with checksum verification
- **Format Normalization**: Automatic handling of hyphens and spaces in ISBNs
- **Error Handling**: Clear error messages for invalid ISBN formats
- **Duplicate Prevention**: Prevents adding the same book twice

### AI Function Calling Integration

The AI assistant can now manage favorites through natural conversation:

```javascript
// Available AI Functions:
-add_to_favorites(isbn13, title) -
  remove_from_favorites(isbn13) -
  list_favorites();
```

**Example Interactions:**

- "Add this book to my favorites"
- "Show me my favorite books"
- "Remove [book title] from my favorites"

### Controller Integration

- **Function Execution**: `executeFavoriteFunction()` handles AI function calls
- **Auto-Clear on Refresh**: Favorites automatically reset on page reload
- **Session Management**: Favorites initialized alongside conversation history

### Data Structure

Each favorite book stores:

- `isbn`: ISBN-13 identifier (normalized, no hyphens)
- `title`: Book title
- `addedAt`: ISO timestamp of when book was added

### Future-Ready Architecture

The favorites service is designed with MongoDB migration in mind:

- Clean data structure ready for database storage
- Session-based implementation allows easy transition
- Timestamp tracking for historical data

## Usage Examples

### Adding to Favorites

```javascript
const result = favoriteService.addFavorite(
  session,
  "9780316769488",
  "The Catcher in the Rye"
);
// { success: true, message: 'Added "The Catcher in the Rye" to your favorites list.' }
```

### Listing Favorites

```javascript
const result = favoriteService.listFavorites(session);
// {
//   success: true,
//   favorites: [...],
//   count: 3,
//   message: 'You have 3 books in your favorites list.'
// }
```

### Removing from Favorites

```javascript
const result = favoriteService.removeFavorite(session, "9780316769488");
// { success: true, message: 'Removed "The Catcher in the Rye" from your favorites list.' }
```

## Migration Notes

### For Developers

When transitioning to MongoDB:

1. Replace session storage with database queries
2. Add user ID field to favorites schema
3. Update validation to check database instead of session
4. Implement proper indexing on ISBN and user ID fields

### For Users

- **Current Behavior**: Favorites reset when page is refreshed
- **Future**: Favorites will persist across sessions once MongoDB is integrated
