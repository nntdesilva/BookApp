# Version 0.11.1 - 2026-03-04

## Testing: Per-Service Test Organization

Migrated all unit tests from the monolith `tests/` directory into each microservice's own `tests/` folder, and added a dedicated `e2e/` folder for live integration tests. CI was rewritten to run only tests relevant to the changed service on each PR.

### Rewritten Tests

Several tests needed rewrites due to architectural changes from the monolith:

| Old test | What changed |
|---|---|
| `authController.test.js` | Rewrote for JWT/bcrypt routes — no more Passport |
| `authMiddleware.test.js` | Rewrote for JWT cookie verification (`jwt.verify`) instead of `req.isAuthenticated()` |
| `bookController.test.js` | Rewrote as `chatRoutes.test.js` using supertest; mocks HTTP clients instead of local service functions |
| `conversationService.test.js` | Fully async; mocks `../config/redis` instead of Express session object |
| `analysisService.test.js` | Mocks `global.fetch` to stub the books-service HTTP call instead of mocking `gutenbergService` directly |
| `favoriteService.test.js` | Rewrote as `favoritesRoutes.test.js`; tests standalone `Favorite` model instead of embedded `User.favorites[]` |

### E2E Integration Tests

Added `e2e/tests/integration.test.js` — live API tests that run against real Gutenberg, OpenAI, and Anthropic endpoints. Tests are imported as modules directly (no running servers needed). A `global.fetch` bridge intercepts inter-service HTTP calls from `analysisService` and routes them through the `gutenbergService` module, so the real Claude Code Execution pipeline is fully exercised.

**14 e2e tests covering:**
- Gutenberg API (search, full text fetch, word count)
- OpenAI embeddings (semantic similarity, batching)
- Claude Code Execution (statistical analysis)
- AI chat service (text response, tool calls, conversation history)
- End-to-end word count verification (AI answer matches computed count)
- End-to-end semantic search (color words in Alice in Wonderland)

Tests skip gracefully when API keys are absent, making them safe to run in environments without secrets.

### Path-Based CI (`ci.yml`)

Replaced the old "run everything on every PR" workflow with a smart per-service pipeline:

```
detect-changes          (dorny/paths-filter — finds which services changed)
    │
    ├── test-auth-service       (only when auth-service/** changes)
    ├── test-favorites-service  (only when favorites-service/** changes)
    ├── test-books-service      (only when books-service/** changes)
    ├── test-analysis-service   (only when analysis-service/** changes)
    ├── test-chat-service       (only when chat-service/** changes)
    ├── test-gateway            (only when gateway/** changes)
    └── test-e2e                (when any service changes)
         │
         └── all-tests-pass     (required gate — PR cannot merge if any job failed)
```

The `all-tests-pass` job uses `if: always()` and checks that all applicable test jobs either succeeded or were skipped — never failed. This is the required branch protection check.

### Service `app.js` Testability

All six service `app.js` files were updated to export the Express app and only bind to a port when run directly (`require.main === module`). This allows supertest to import the app without starting a server, keeping unit tests fast and isolated.
