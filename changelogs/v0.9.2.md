# Version 0.9.2 - 2026-02-17

## Optimization: Condensed Analysis Prompts for Token Efficiency

The `CODE_EXECUTION_SYSTEM` and `VISUALIZATION_SYSTEM` prompts in `analysisService.js` have been condensed for token efficiency. All behavioural rules, design specifications, and constraints are fully preserved — the prompts are denser but functionally identical.

## What's New

### Prompt Condensation

Both system prompts were rewritten to use more concise language while retaining every rule and constraint:

| Prompt | Purpose |
|---|---|
| `CODE_EXECUTION_SYSTEM` | Instructs Claude how to write and execute Python code for book text analysis |
| `VISUALIZATION_SYSTEM` | Instructs Claude how to generate self-contained Plotly.js HTML visualizations |

### Why

- **Lower token usage** — fewer input tokens per API call reduces cost and latency
- **Same behaviour** — no functional changes; Claude follows the same rules and produces the same output
- **Better maintainability** — tighter prompts are easier to scan and update

## Files Changed

- **`services/analysisService.js`** — rewritten `CODE_EXECUTION_SYSTEM` and `VISUALIZATION_SYSTEM` prompt strings

---

All existing features are unchanged. Analysis and visualization output is identical to v0.9.1.
