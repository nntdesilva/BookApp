# Version 0.11.2 - 2026-03-11

## Bug Fix: Gateway Auth Debug Logging + Service URL Fixes

Two separate fixes shipped together: diagnostic logging added to the gateway login/signup routes to surface auth failures in CloudWatch, and a root-cause fix for those failures caused by a trailing space in `AUTH_SERVICE_URL` and missing inter-service URLs across multiple App Runner services.

## What's New

### Debug Logging in Gateway Login / Signup Routes

Added structured logging to `gateway/app.js` to make auth failures visible in CloudWatch:

- Every `POST /login` and `POST /signup` attempt logs the target auth-service URL before the fetch
- Fetch-level errors (network unreachable, DNS failure) are logged with `fetchErr.message` and `fetchErr.cause`
- Response body parse errors are logged with the raw HTTP status
- The auth-service HTTP status, `success` flag, and `error` field are logged for every response
- All service URLs (`authUrl`, `chatUrl`, `favoritesUrl`) are logged at gateway startup so misconfigured env vars appear immediately

### Trailing-Space Fix for Service URLs

`AUTH_SERVICE_URL` on AWS App Runner had a trailing space, causing every constructed URL (`${AUTH_SERVICE_URL}/api/auth/login`) to resolve to an invalid address. Added `.trim()` to all service URL reads:

- `gateway/config/appConfig.js` — `AUTH_SERVICE_URL`, `CHAT_SERVICE_URL`, `FAVORITES_SERVICE_URL`
- `chat-service/config/appConfig.js` — all service URL reads
- `analysis-service/config/appConfig.js` — all service URL reads

### Missing Inter-Service URLs on App Runner

Several App Runner services were falling back to `localhost` defaults because their environment variables were never set:

| Service | Missing variables |
|---|---|
| gateway | `CHAT_SERVICE_URL`, `FAVORITES_SERVICE_URL` |
| chat-service | `FAVORITES_SERVICE_URL`, `BOOKS_SERVICE_URL`, `ANALYSIS_SERVICE_URL` |
| analysis-service | `BOOKS_SERVICE_URL` |

All three App Runner services were updated directly with the correct production URLs.

## Files Changed

- **`gateway/app.js`** — added fetch-error, parse-error, response-status, and startup URL logs to login and signup routes
- **`gateway/config/appConfig.js`** — added `.trim()` to all service URL env var reads
- **`chat-service/config/appConfig.js`** — added `.trim()` to all service URL env var reads
- **`analysis-service/config/appConfig.js`** — added `.trim()` to all service URL env var reads
