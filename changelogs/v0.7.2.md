# Version 0.7.2 - 2026-02-08

## New Feature: Related Word Count

This release introduces semantic word search using embeddings. Users can now find and count words that are contextually related to a concept, not just exact matches.

## What's New

### Semantic Word Search

- **Embedding-Based Analysis**: Uses OpenAI embeddings to find semantically similar words
- **Concept Matching**: Search for words related to concepts (e.g., "emotion", "weather", "violence")
- **Similarity Scoring**: Each related word includes a similarity score
- **Contextual Understanding**: Goes beyond exact word matching to find related terms

### Embedding Service

Created `services/embeddingService.js` with the following features:

- **Text Embeddings**: Generate embeddings using OpenAI's `text-embedding-3-small` model
- **Batch Processing**: Efficiently process multiple words in batches
- **Cosine Similarity**: Calculate semantic similarity between concept and words
- **Configurable Thresholds**: Adjustable similarity thresholds and result limits

### Related Word Counting

- **Full Book Analysis**: Extracts all unique words from book text
- **Semantic Filtering**: Finds words semantically related to the concept
- **Occurrence Counting**: Counts each related word in the book
- **Sorted Results**: Results sorted by occurrence count
- **Detailed Output**: Returns word, count, and similarity score for each match

## Implementation Details

### Embedding Pipeline

1. **Text Extraction**: Get full book text from Project Gutenberg
2. **Word Extraction**: Extract unique words using regex
3. **Embedding Generation**: Create embeddings for concept and words (batched)
4. **Similarity Calculation**: Compute cosine similarity for each word
5. **Filtering**: Keep words above similarity threshold
6. **Sorting**: Rank by similarity score
7. **Counting**: Count occurrences of each related word
8. **Aggregation**: Sum total occurrences and provide statistics

### Cosine Similarity Formula

```javascript
function cosineSimilarity(vecA, vecB) {
  const dotProduct = vecA.reduce((sum, a, i) => sum + a * vecB[i], 0);
  const magnitudeA = Math.sqrt(vecA.reduce((sum, a) => sum + a * a, 0));
  const magnitudeB = Math.sqrt(vecB.reduce((sum, b) => sum + b * b, 0));
  return dotProduct / (magnitudeA * magnitudeB);
}
```

- Handles large vocabularies efficiently

## Configuration

Tunable parameters in `config/appConfig.js`:

```javascript
embedding: {
  model: "text-embedding-3-small",  // OpenAI embedding model
  similarityThreshold: 0.5,          // Minimum similarity to include
  maxRelatedWords: 20,               // Maximum words to return
  batchSize: 100                     // Words per API request
}
```

## Troubleshooting

### High API Costs

- Reduce `maxRelatedWords` in config
- Increase `similarityThreshold` to get fewer results
- Implement caching for frequently analyzed books

### Unexpected Results

- Adjust `similarityThreshold` (0.5 is default)
- Try more specific concepts
- Check embedding model documentation for behavior

### Slow Performance

- Reduce vocabulary size by preprocessing
- Increase batch size (within API limits)
- Consider caching unique words per book
