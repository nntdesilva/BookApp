# Version 0.11.0 - 2026-03-04

## Architecture: Monolith to Microservices

The entire application has been decomposed from a single Express.js monolith into seven independently deployable microservices, orchestrated via Docker Compose.

## What Changed

### Architecture Overview

The monolith (`app.js` + `controllers/` + `services/` + `routes/`) has been replaced by:

| Service | Port | Responsibility |
|---|---|---|
| **gateway** | 3000 | Serves EJS views, proxies API calls to backend services, JWT cookie management |
| **auth-service** | 3001 | User registration, login, JWT issuance, dark mode preference |
| **favorites-service** | 3002 | CRUD for book favorites (standalone MongoDB collection) |
| **books-service** | 3003 | Gutenberg API search, full-text fetching, OpenAI embeddings, word counting |
| **analysis-service** | 3004 | Claude Code Execution for text analysis and Plotly.js visualization |
| **chat-service** | 3005 | AI orchestrator ‚Äî agentic tool-use loop, conversation history (Redis) |
| **mcp-server** | 3006 | MCP tool interface for IDE integration (calls services via HTTP) |

### Authentication: Sessions ‚Üí JWT + Redis

| Before | After |
|---|---|
| Passport.js + express-session + connect-mongo | JWT tokens in httpOnly cookies |
| Session-stored conversation history | Redis-backed conversation history (`conv:{userId}`) |
| Session-stored visualization cache | Redis-backed analysis cache in analysis-service |
| `passport-local-mongoose` plugin | `bcrypt` for password hashing, `jsonwebtoken` for token signing |

### Data Model Changes

**User model** ‚Äî split across services:
- `auth-service/models/User.js` ‚Äî credentials (`username`, `passwordHash`, `email`, `darkMode`) using `bcrypt` instead of `passport-local-mongoose`
- `favorites-service/models/Favorite.js` ‚Äî standalone collection (`userId`, `isbn`, `title`, `addedAt`) instead of embedded sub-document in User

### Inter-Service Communication

All services communicate via HTTP. The chat-service is the orchestrator that drives Claude's agentic tool-use loop and dispatches tool calls to other services via HTTP clients:

- `chat-service/clients/favoritesClient.js` ‚Üí favorites-service
- `chat-service/clients/booksClient.js` ‚Üí books-service
- `chat-service/clients/analysisClient.js` ‚Üí analysis-service

The gateway proxies browser requests to the appropriate backend service, injecting `x-user-id` and `x-username` headers after JWT verification.

### Infrastructure

- **Docker Compose** (`docker-compose.yml`) orchestrates all seven services plus MongoDB and Redis
- Each service has its own `Dockerfile`, `package.json`, and configuration
- Services use separate MongoDB databases (`bookapp_auth`, `bookapp_favorites`)
- Redis is shared between chat-service (conversation state) and analysis-service (visualization cache)

### MCP Server

The MCP server (`mcp-server/`) no longer imports service files directly or connects to MongoDB. It calls the microservices via HTTP, making it a thin adapter layer that can be deployed independently.

## How to Run

```bash
# Docker Compose (recommended)
cp .env.example .env    # fill in API keys
docker compose up --build

# Local development (run each service separately)
npm run install:all     # install deps for all services
npm run dev:auth        # terminal 1
npm run dev:favorites   # terminal 2
npm run dev:books       # terminal 3
npm run dev:analysis    # terminal 4
npm run dev:chat        # terminal 5
npm run dev:gateway     # terminal 6
```

## Files Removed

All monolith files have been removed from the root:
- `app.js`, `mcp-server.js`, `Dockerfile`, `Dockerfile.mcp`
- `controllers/`, `services/`, `routes/`, `middleware/`, `models/`, `config/`, `utils/`
- `views/`, `public/` (moved to `gateway/`)
