# Version 0.11.4 - 2026-03-11

## Bug Fix: Missing RuntimeEnvironmentVariables in CD Pipeline for All Services

Fixed all services returning runtime errors in production caused by the CD pipeline deploying every service with an `ImageConfiguration` that only set the `Port`. Because no `RuntimeEnvironmentVariables` block was present in the `update-service` call, App Runner wiped every environment variable on each deployment, so secrets were never injected into running containers.

## What's New

### Root Cause

The `aws apprunner update-service` calls in `.github/workflows/cd.yml` only specified `Port` inside `ImageConfiguration`. App Runner interprets a missing `RuntimeEnvironmentVariables` key as "clear all env vars", so every CD deployment silently removed all secrets from the running service.

### Variables Added per Service

| Service | Variables added |
|---|---|
| gateway | `JWT_SECRET`, `AUTH_SERVICE_URL`, `CHAT_SERVICE_URL`, `FAVORITES_SERVICE_URL`, `NODE_ENV` |
| auth-service | `JWT_SECRET`, `MONGODB_URI`, `NODE_ENV` |
| favorites-service | `JWT_SECRET`, `MONGODB_URI`, `NODE_ENV` |
| books-service | `OPENAI_EMBEDDINGS_API_KEY`, `MONGODB_URI`, `NODE_ENV` |
| analysis-service | `ANTHROPIC_API_KEY`, `REDIS_URL`, `BOOKS_SERVICE_URL`, `NODE_ENV` |
| chat-service | `ANTHROPIC_API_KEY`, `REDIS_URL`, `FAVORITES_SERVICE_URL`, `BOOKS_SERVICE_URL`, `ANALYSIS_SERVICE_URL`, `NODE_ENV` |
| mcp-server | `FAVORITES_SERVICE_URL`, `BOOKS_SERVICE_URL`, `ANALYSIS_SERVICE_URL`, `NODE_ENV` |

### Pre-Flight Secret Checks

Updated each service's "Verify required secrets" step to enumerate every variable that service needs. The job now exits with a non-zero code listing all missing secrets before any AWS calls are made, making misconfiguration immediately visible in the Actions log rather than producing a silently broken deployment.

## Files Changed

- **`.github/workflows/cd.yml`** — added `RuntimeEnvironmentVariables` block and expanded pre-flight secret checks for all seven `deploy-*` jobs
