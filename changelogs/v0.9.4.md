# Version 0.9.4 - 2026-02-18

## Testing: Comprehensive Jest Unit Test Suite

Added a full unit test suite covering all services, controllers, middleware, and utilities. 198 tests across 11 test files, all located in the `tests/` directory. External dependencies (Claude, OpenAI, MongoDB, Gutenberg) are fully mocked.

## What's New

### Test Coverage

| `analysisService` | Service layer | Book statistics analysis, visualization generation, Claude Code Execution |

| `utils` | Utilities | Helper functions |

### Test Scenarios

Each test file covers:

- **Happy path** — expected inputs produce correct outputs
- **Edge cases** — empty inputs, boundary values, missing optional fields
- **Error/failure scenarios** — malformed LLM output, empty responses, token limit errors, network failures

### Mocking Strategy

All external dependencies are fully mocked:

- **Anthropic SDK** — Claude chat responses and Code Execution results
- **OpenAI SDK** — embedding vectors and API responses
- **Mongoose/MongoDB** — database queries and model operations
- **HTTP requests** — Gutenberg/Gutendex API responses

## Files Added

- **`tests/`** — 11 test files with 198 total tests

## Running the Tests

```bash
# Run all unit tests
npx jest --verbose --testPathIgnorePatterns="tests/integration.test.js"

# Run with coverage
npx jest --coverage --testPathIgnorePatterns="tests/integration.test.js"
```

All existing features are unchanged.
