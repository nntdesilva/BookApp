# Version 0.7.0 - 2026-02-07

## Major Feature: Word Count in Books

This release introduces powerful word search functionality using Project Gutenberg integration. Users can now ask the AI to count word occurrences in classic literature books, with the AI interpreting queries and searching full book texts.

## What's New

### Project Gutenberg Integration

- **Full Text Access**: Fetch complete book texts from Project Gutenberg
- **Gutendex API**: Search for books using the Gutendex metadata API
- **Text Processing**: Extract and analyze full book content
- **Author Metadata**: Retrieve author information alongside book data

### Word Count Functionality

- **Natural Language Queries**: Ask questions like "How many times does 'whale' appear in Moby Dick?"
- **AI Query Interpretation**: AI understands intent and extracts book title and search term
- **Precise Counting**: Exact word occurrence counting using regex
- **Case-Insensitive Search**: Matches words regardless of capitalization

### Gutenberg Service

Created `services/gutenbergService.js` with the following capabilities:

- **Book Search**: Find books by title using Gutendex API
- **Full Text Retrieval**: Download complete book texts from Project Gutenberg
- **Word Counting**: Regex-based word occurrence counting
- **Unique Word Extraction**: Extract all unique words from book text for analysis
- **Title Resolution**: Resolve ambiguous book titles to specific editions

### AI Function Calls

New AI functions for word search:

- `resolve_book_for_search`: Find and confirm book availability
- `count_word_in_book`: Count specific word occurrences in a book

## Usage Examples

### Word Count Query

```
User: "How many times does 'Uriah' appear in David Copperfield?"

AI Process:
1. Interprets query → book: "David Copperfield", word: "Uriah"
2. Calls resolve_book_for_search("David Copperfield")
3. Calls count_word_in_book("David Copperfield", "Uriah")
4. Returns: "The word 'Uriah' appears 253 times in David Copperfield by Charles Dickens."
```

## Implementation Details

### Word Counting Algorithm

```javascript
// Case-insensitive regex word boundary matching
const regex = new RegExp(`\\b${word}\\b`, 'gi');
const matches = text.match(regex);
const count = matches ? matches.length : 0;
```

### Book Text Retrieval

1. Search Gutendex API for book metadata
2. Extract text download URL (format: text/plain; charset=utf-8)
3. Fetch full text from Project Gutenberg mirror
4. Process and return with metadata

## Troubleshooting

### Common Issues

**Book Not Found**:
- Verify exact book title
- Try alternative titles or author names
- Check if book is in Project Gutenberg catalog

**Slow Response**:
- First query downloads full book text (large files)
- Network speed affects download time
- Consider implementing caching

**Word Count Seems Wrong**:
- Check for variations (capitalization, punctuation)
- Regex uses word boundaries (whole words only)
- OCR errors in source text may affect accuracy

## Architecture Notes

### Service Layer

```
User Query → AI Service → Gutenberg Service → Project Gutenberg
                ↓              ↓                      ↓
         Function Calls   API Requests         Text Download
                ↓              ↓                      ↓
         Execute Functions → Word Counting    → Results
                ↓
         Format Response → Return to User
```

### Data Flow

1. User asks word count question
2. AI parses query and identifies book + word
3. Function calls: resolve book, count word
4. Gutenberg service fetches and processes text
5. Results formatted and returned to user
