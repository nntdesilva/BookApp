# Version 0.11.14 - 2026-03-24

## Refactor: LangChain in chat-service

The chat-service **AI layer** and **conversation persistence** now use [LangChain](https://js.langchain.com/) instead of calling the Anthropic SDK directly.

- **`aiService.js`:** `ChatAnthropic` from `@langchain/anthropic` with `bindTools()`; tools are declared with Zod via `tool()` from `@langchain/core/tools`. The system prompt is wrapped in `ChatPromptTemplate`. Messages use `HumanMessage` and `ToolMessage` from `@langchain/core/messages`. Model `maxRetries` replaces the previous hand-rolled retry loop.
- **`conversationService.js`:** Per-user history is stored with `RedisChatMessageHistory` from `@langchain/community` (session keys `conv:{userId}`, 24-hour TTL).
- **`config/redis.js`:** The in-memory Redis fallback implements list operations (`lrange`, `rpush`, `lpush`, `expire`, `ltrim`) so LangChain’s history works when Redis is unavailable; the module exposes a `client` getter used by the message store.

Dependencies: `@langchain/anthropic`, `@langchain/community`, `@langchain/core`, and `zod`. Unit tests were updated for the new message types and APIs.
