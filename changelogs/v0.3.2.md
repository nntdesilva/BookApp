# Version 0.3.2 - 2026-01-26

## Fixed

### Issue 1: Standalone Book Title Badging Consistency
- **Problem**: Standalone books (e.g., "The 48 Laws of Power", "1984") were not being consistently badged in initial search results
- **Root Cause**: System prompt lacked explicit instructions for wrapping book titles on first mention
- **Solution**: Enhanced system prompt with:
  - Stronger directive: "CRITICAL: You MUST wrap every book title"
  - Explicit instruction: "ALWAYS wrap the main book being discussed - even when first introducing it"
  - Concrete good vs bad examples for clarity
  - Specific example: "[[The 48 Laws of Power]] is a non-fiction book by Robert Greene..."
- **Result**: Achieved 100% consistent badging for all book types (series, standalone, bibliographies)
- **Testing**: Verified with "The 48 Laws of Power", "1984", and Robert Greene bibliography queries

### Issue 2: Series Name Search Returning Incomplete Titles
- **Problem**: When searching for a series name (e.g., "Harry Potter"), only "Harry Potter" was being badged instead of the full book title
- **Root Cause**: Title identification step returned series name instead of first book's complete title
- **Impact**: Series name got badged (incorrect) instead of actual book title
- **Solution**: Enhanced title identification prompt to:
  - Recognize series names and character names
  - Return the FIRST book's COMPLETE title when given a series name
  - Explicit instruction: "If input is a series name, return the FIRST book's complete title"
  - Added concrete examples: "Harry Potter" → "Harry Potter and the Philosopher's Stone"
- **Result**: Correct behavior for series searches - full book title gets badged, series name does not
- **Testing**: Verified with "Harry Potter", "Lord of the Rings" searches
