# Version 0.9.1 - 2026-02-17

## Migration: Claude Models for All AI Features (except Embeddings)

This release migrates the primary AI provider from OpenAI GPT to Anthropic Claude across all AI-powered features. OpenAI is retained exclusively for semantic word search embeddings (`text-embedding-3-large`), where it continues to perform best.

## What's New

### Claude as Primary AI Provider

- **Chat & Tool Use**: `aiService.js` fully rewritten using `@anthropic-ai/sdk` with Claude's Messages API
- **Book Statistics & Visualization**: `analysisService.js` (formerly `assistantService.js`) migrated from OpenAI Assistants API to Claude's Code Execution beta tool
- **Tool Format**: Tool definitions updated from OpenAI function format to Anthropic's `input_schema` format
- **Conversation History**: Session history now stores Claude-native content blocks for full multi-turn compatibility

### OpenAI Retained for Embeddings Only

- `embeddingService.js` continues using OpenAI `text-embedding-3-large` for semantic word search
- A dedicated `OPENAI_EMBEDDINGS_API_KEY` environment variable separates embedding credentials from general AI credentials
- All other features are fully decoupled from OpenAI

## Technical Details

- **`controllers/bookController.js`**:
  - Updated to handle Claude's `tool_use` block format for function dispatch
  - `tool_result` responses built using Claude's content block structure

### Claude Code Execution Pipeline

Replaces OpenAI's thread/run/poll pattern with Claude's stateful container approach:

```
analyzeBookStatistics():
  1. Fetch book text from Project Gutenberg
  2. Upload book.txt via Anthropic Files API (beta)
  3. Send container_upload + user question to Claude
  4. Claude writes and executes Python in sandboxed container
  5. Handle pause_turn continuations if needed (up to 5)
  6. Extract text from all response blocks
  7. Delete uploaded file (finally block)

generateVisualization():
  1. Receive pre-computed analysis data
  2. Claude writes Python to build self-contained Plotly.js HTML
  3. HTML extracted between ---HTML_START--- and ---HTML_END--- markers
  4. Handle pause_turn continuations if needed (up to 3)
  5. Reuse container across continuations via container.id
```

### API Configuration

```javascript
// Claude (primary AI)
claude: {
  apiKey: process.env.ANTHROPIC_API_KEY,
  model: process.env.CLAUDE_MODEL || "claude-sonnet-4-20250514",
  temperature: parseFloat(process.env.CLAUDE_TEMPERATURE) || 0.7,
  maxTokens: parseInt(process.env.CLAUDE_MAX_TOKENS) || 1000,
}

// OpenAI (embeddings only)
embedding: {
  apiKey: process.env.OPENAI_EMBEDDINGS_API_KEY,
  model: process.env.EMBEDDING_MODEL || "text-embedding-3-large",
  similarityThreshold: 0.55,
  batchSize: 2048,
}
```

## Benefits

1. **Unified AI Provider**: Claude handles all reasoning, tool use, and code execution
2. **Better Tool Integration**: Claude's native tool use is tightly integrated with the agentic meta-prompt
3. **Stateful Code Execution**: Claude's container reuse across `pause_turn` continuations avoids re-uploading files
4. **Cleaner Architecture**: OpenAI dependency scoped exclusively to the embedding layer
5. **Configurable Model**: Claude model swappable via `CLAUDE_MODEL` environment variable

## Migration Notes

When upgrading from v0.9.0:

1. Add `ANTHROPIC_API_KEY` to your environment
2. Rename any existing `OPENAI_API_KEY` to `OPENAI_EMBEDDINGS_API_KEY` (or add the new variable separately)
3. Optionally set `CLAUDE_MODEL` to override the default model
4. The `openai` package must remain installed â€” it is still required for embeddings

All existing features (book search, favorites, word count, semantic search, book statistics, visualization) continue to work identically from the user's perspective.
