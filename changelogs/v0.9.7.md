# Version 0.9.7 - 2026-02-23

## Migration: GCP Cloud Run → AWS App Runner

This release migrates the deployment platform from Google Cloud Platform (Cloud Run + Cloud Build + Artifact Registry) to Amazon Web Services (App Runner + ECR). The CI/CD pipeline, container configuration, and health check infrastructure have all been updated accordingly.

## What's New

### Deployment pipeline rewritten for AWS (`ci.yml`)

- Replaced GCP Workload Identity Federation auth with AWS OIDC via `aws-actions/configure-aws-credentials`
- Replaced Cloud Build submission with direct Docker build/push to Amazon ECR
- Replaced Cloud Run deploy with `aws apprunner update-service`, including HTTP health check configuration on `/health`
- Removed the async Cloud Build polling loop — App Runner uses `aws apprunner wait service-running` instead
- Job renamed from "Deploy to GCP Cloud Run" to "Deploy to AWS App Runner"

### Health check endpoint (`app.js`)

- Added `GET /health` that returns `{ "status": "ok" }` with a 200 response
- Placed before all other middleware (session store, Passport, routes) so it never depends on MongoDB or any external service
- App Runner probes this endpoint to verify container liveness

### Non-fatal MongoDB connection (`config/database.js`)

- Replaced `process.exit(1)` on connection failure with a warning log
- The server continues running when MongoDB is unreachable, preventing crash loops during cold starts
- Required for App Runner health checks to pass while the database is still connecting

### Dockerfile update

- Added `ENV PORT=8080` to explicitly set the port environment variable for App Runner

### Removed GCP-specific files

- Deleted `cloudbuild.yaml` (no longer needed — Docker builds now happen directly in the GitHub Actions workflow)

## Required GitHub Secrets

| Old (GCP) | New (AWS) |
|---|---|
| `GCP_WIF_PROVIDER` | `AWS_ROLE_ARN` |
| `GCP_WIF_SERVICE_ACCOUNT` | `AWS_REGION` |
| `GCP_PROJECT_ID` | `AWS_ACCOUNT_ID` |
| — | `APPRUNNER_SERVICE_ARN` |

## Deployment Flow

```
Push / merge to main
  → test job (unit tests + integration tests)
    → deploy job (only if tests pass)
      → AWS OIDC auth → ECR login
        → Docker build & push to ECR
          → App Runner update-service → wait for stable
```

---

All existing features and tests are unchanged.
