# Version 0.9.6 - 2026-02-20

## CI/CD: Continuous Deployment Pipeline to GCP Cloud Run

This release adds a continuous deployment pipeline using GitHub Actions. Merging to `main` now automatically builds, pushes, and deploys the app to Google Cloud Run — but only after all tests pass.

## What's New

### GitHub Actions Workflow (`ci.yml`)

The existing CI workflow has been extended into a full CI/CD pipeline.

#### Trigger changes

- Previously only triggered on pull requests to `main` and `dev`
- Now also triggers on **push to `main`** (i.e. when a PR is merged), which kicks off the deployment job

#### New `deploy` job

- Runs only on push to `main` (`github.event_name == 'push' && github.ref == 'refs/heads/main'`)
- Depends on the `test` job via `needs: test` — deployment is blocked if any test fails
- Authenticates to GCP using **Workload Identity Federation** (keyless, no long-lived service account keys stored as secrets)
- Submits `cloudbuild.yaml` to Cloud Build with the current commit SHA as a substitution variable

## Technical Details

### Files Changed

- **`.github/workflows/ci.yml`**: Added `push` trigger on `main` and a new `deploy` job with 4 steps: checkout, GCP auth, Cloud SDK setup, and Cloud Build submission

### Files Added

- **`Dockerfile`**: Defines the container image for the app
- **`.dockerignore`**: Excludes `node_modules`, `.env`, and other non-essential files from the Docker build context
- **`cloudbuild.yaml`**: Cloud Build pipeline — builds and tags the Docker image, pushes it to Artifact Registry (`us-central1-docker.pkg.dev`), and deploys to Cloud Run in `us-central1`

### Required GitHub Secrets

| Secret | Description |
|---|---|
| `GCP_PROJECT_ID` | GCP project ID |
| `GCP_WIF_PROVIDER` | Workload Identity Federation provider resource name |
| `GCP_WIF_SERVICE_ACCOUNT` | Service account email used for deployment |

### Deployment Flow

```
Push / merge to main
  → test job (unit tests + integration tests)
    → deploy job (only if tests pass)
      → gcloud builds submit
        → Docker build → Artifact Registry push → Cloud Run deploy
```

---

All existing features and tests are unchanged.
