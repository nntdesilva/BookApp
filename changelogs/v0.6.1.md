# Version 0.6.1 - 2026-02-04

## Major Update: MongoDB Storage & User Authentication

This release transforms the BookApp from a session-based application to a fully authenticated, database-backed system. Users can now create accounts, log in securely, and have their favorites persist across sessions.

## What's New

### MongoDB Integration

- **Database Configuration**: Implemented `config/database.js` for MongoDB connection management
- **Connection Pooling**: Efficient database connection handling with automatic reconnection
- **Session Store**: Migrated from in-memory sessions to MongoDB-backed persistent sessions using `connect-mongo`
- **Environment Variables**: Added MongoDB URI configuration via `.env` file

### User Authentication with Passport.js

- **Local Strategy**: Username/password authentication using `passport-local`
- **Secure Password Storage**: BCrypt password hashing with salt rounds (10)
- **Session Management**: User sessions persisted across server restarts
- **Passport Configuration**: Implemented `config/passport.js` with serialization/deserialization

### User Model

Created `models/User.js` with the following features:

- **Schema Fields**:
  - `username`: Unique, required, lowercase
  - `email`: Unique, required, valid email format
  - `password`: BCrypt-hashed, never exposed to client
  - `favorites`: Array of book objects with ISBN, title, and timestamp
  - `createdAt`: Account creation timestamp

- **Password Security**:
  - Pre-save hook for automatic password hashing
  - `comparePassword()` method for secure login verification
  - Passwords never stored in plain text

### Authentication Middleware

- **Protected Routes**: `middleware/auth.js` ensures user authentication
- **Redirect Logic**: Unauthenticated users redirected to login page
- **Session Validation**: Checks for valid Passport session

### Authentication Controller

Created `controllers/authController.js` with handlers for:

- `renderSignup()`: Display signup form
- `handleSignup()`: Process new user registration
- `renderLogin()`: Display login form
- `handleLogin()`: Process user authentication
- `handleLogout()`: Clear session and redirect

### Authentication Routes

- **GET `/auth/signup`**: Signup page
- **POST `/auth/signup`**: User registration endpoint
- **GET `/auth/login`**: Login page
- **POST `/auth/login`**: Authentication endpoint
- **GET `/auth/logout`**: Logout and session destruction

### User Interface

Created authentication views:

- `views/auth/login.ejs`: Clean login form with validation
- `views/auth/signup.ejs`: User registration form with email validation
- **Navigation Updates**: Added user info display and logout button in header
- **Responsive Design**: Mobile-friendly authentication forms

### Favorites Service Enhancement

Updated `services/favoriteService.js` for database integration:

- **User-Scoped Favorites**: Each user has their own favorites collection
- **Persistent Storage**: Favorites saved to MongoDB, survive server restarts
- **Atomic Operations**: Use MongoDB `$push`, `$pull` for reliable updates
- **Migration Ready**: Seamless transition from session-based to DB-based storage

## Setup Instructions

### Prerequisites

- MongoDB installed locally or MongoDB Atlas account
- Node.js and npm

### Installation Steps

1. Install MongoDB (if not already installed)
2. Start MongoDB service:
   ```bash
   mongod
   ```
3. Copy `.env.example` to `.env` and configure:
   ```bash
   cp .env.example .env
   ```
4. Update `.env` with your MongoDB connection string
5. Install dependencies:
   ```bash
   npm install
   ```
6. Start the application:
   ```bash
   npm start
   ```
7. Navigate to `http://localhost:3000/auth/signup` to create an account

## Troubleshooting

### Common Issues

**MongoDB Connection Error**:
- Ensure MongoDB service is running
- Check `MONGODB_URI` in `.env` file
- Verify network connectivity for cloud databases

**Login Fails**:
- Verify username/password are correct
- Check server logs for detailed error messages
- Ensure BCrypt is properly installed

**Session Not Persisting**:
- Verify `SESSION_SECRET` is set in `.env`
- Check MongoDB connection for session store
- Clear browser cookies and try again

## Architecture Notes

### Database Schema

```javascript
User {
  username: String (unique, indexed),
  email: String (unique, indexed),
  password: String (hashed),
  favorites: [{
    isbn: String,
    title: String,
    addedAt: Date
  }],
  createdAt: Date
}
```

### Authentication Flow

1. User submits credentials
2. Passport.js validates against database
3. BCrypt compares hashed passwords
4. Session created and stored in MongoDB
5. User redirected to main application

### Session Management

- Sessions stored in MongoDB `sessions` collection
- 24-hour TTL (configurable)
- Automatic cleanup of expired sessions
- Secure cookie configuration
