# Version 0.9.5 - 2026-02-18

## Testing: Integration Test Suite with Live API Calls

This release adds a real-API integration test suite that verifies every major feature end-to-end against live Gutenberg, OpenAI, and Anthropic services. Unlike the existing unit tests (which mock all external dependencies), these tests make actual network calls to confirm the full stack produces correct results.

## What's New

### Integration Test File

A new file `tests/integration.test.js` covers all service layers using real credentials from `.env`.

#### Gutenberg Service (no API key required)

- Searches and resolves "Pride and Prejudice", verifying title and author metadata
- Fetches the full text of "A Christmas Carol" and checks it exceeds 10,000 characters
- Counts the word "elizabeth" in *Pride and Prejudice* and asserts the result exceeds 500
- Cross-validates that `countWordOccurrences()` and `countWordInBook()` return identical counts for "darcy"
- Asserts a clean failure response for a book title that does not exist in Gutenberg

#### Embedding Service (requires `OPENAI_EMBEDDINGS_API_KEY`)

- Confirms semantically related words for the concept "love" are found (romance, heart, passion, etc.) while unrelated words (computer, refrigerator) are excluded
- Verifies that results are ordered by cosine similarity descending
- Confirms an empty result when the similarity threshold is set impossibly high (0.95)

#### Analysis Service (requires `ANTHROPIC_API_KEY`)

- Asks Claude Code Execution to count "Scrooge" in *A Christmas Carol*, then cross-checks the result against the local regex counter, asserting the difference is within ±15 (to account for Gutenberg header/footer variation)

#### AI Chat Service (requires `ANTHROPIC_API_KEY`)

- Sends a general book question and verifies a non-empty text response is returned with correct conversation history length
- Asks a word-count question and asserts Claude requests a `resolve_book_for_search` or `count_word_in_book` tool call
- Verifies multi-turn memory: asks the AI to remember a number across two conversation turns

#### End-to-End Word Count Verification (requires `ANTHROPIC_API_KEY`)

- Independently computes the exact count of "pride" in *Pride and Prejudice* using `gutenbergService`
- Sends the same question to the AI, manually drives all tool-call rounds (same loop as `bookController`), and asserts the AI's final natural-language response contains the exact count number

#### End-to-End Semantic Search (requires both API keys)

- Fetches *Alice's Adventures in Wonderland*, extracts unique words, runs embedding-based concept search for "colors", and verifies at least 2 actual color words (white, red, golden, etc.) are returned

## Technical Details

### New File

- **`tests/integration.test.js`**:
  - 14 tests across 6 `describe` blocks
  - `jest.setTimeout(180_000)` — 3-minute per-test timeout to accommodate Gutenberg fetches, Files API uploads, and multi-round tool calls
  - Tests that require API keys use `test.skip` automatically when the corresponding key is absent from the environment
  - No mocks — all external calls are live

### Running the Tests

```bash
# Run all integration tests
npx jest tests/integration.test.js --verbose

# Run a specific group only
npx jest tests/integration.test.js --verbose -t "Gutenberg Service"
npx jest tests/integration.test.js --verbose -t "End-to-End Word Count"
```

Requires `ANTHROPIC_API_KEY` and `OPENAI_EMBEDDINGS_API_KEY` to be set in `.env`. Tests that need keys are automatically skipped if the key is missing.

Expected runtime: **60–120 seconds** for the full suite.
